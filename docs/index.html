Fast SSH/SCP file transfer tool with named targets and path aliases

Installation

Install from crates.io: 
cargo install quick-copy

This installs both quick-copy and qc (short alias).

Quick Start

1. Create configuration file

Create ~/.config/quick-copy/config.yaml:

version: 1

defaults:
  user: myuser
  port: 22
  default_path_alias: scratch
  staging_dir: "/tmp"
  zip:
    exclude:
      - ".git/"
      - "node_modules/"

hosts:
  server:
    host: "192.168.1.100"
    paths:
      scratch: "/home/myuser/scratch"
      projects: "/home/myuser/projects"

  nas:
    host: "nas.local"
    paths:
      shared: "/mnt/shared"

shares:
  default: "nas:shared"
  layout: "{source}/{date}"

2. Copy files

qc file model.pt server:scratch
qc dir server:projects
qc pull server:scratch
qc dump ./results

Commands

Copy a file

qc file report.pdf server:scratch
qc file data.csv server:/absolute/path
qc file --dry-run model.pt server:scratch

Copies a single file to a remote destination.

Copy current directory

qc dir server:scratch
qc dir server:projects --name backup.zip
qc dir server:scratch --exclude "*.log" --exclude "data/"
qc dir --extract server:scratch

Zips and copies the current directory. Excludes patterns from config are applied automatically.

Pull remote directory

qc pull server:scratch
qc pull server:/remote/path
qc pull server:projects --no-extract

Downloads a remote directory to your current working directory. Extracts by default.

Dump to shared storage

qc dump ./results
qc dump model.pt
qc dump

Copy files to shared storage (configured in shares.default). Organizes by source hostname and date.

List targets

qc ls

Shows configured hosts and path aliases.

Check setup

qc doctor
qc doctor --test server nas

Validates configuration and SSH connectivity.

Preview operations (dry-run)

qc --dry-run file report.pdf server:scratch
qc --dry-run dir server:projects
qc --dry-run pull server:data

Preview what would happen without executing the transfer.

Configuration

Destination Syntax

Format Description Example
host Use host's default path alias qc file data.csv server
host:alias Use named path alias qc file data.csv server:scratch
host:/path Use explicit absolute path qc file data.csv server:/tmp

Configuration Locations

quick-copy searches for configuration in this order:

  1. Path specified with -c/--config flag
  2. .quick-copy.yaml in current directory
  3. ~/.config/quick-copy/config.yaml
  4. ~/.quick-copy.yaml

Host Configuration

hosts:
  workstation:
    host: "workstation.local"
    role: "development"
    paths:
      scratch: "/home/myuser/scratch"
      projects: "/home/myuser/projects"

  server:
    host: "192.168.1.100"
    port: 2222                    # Custom SSH port
    user: "admin"                 # Override default user
    identity_file: "~/.ssh/id_rsa_server"
    paths:
      scratch: "/home/admin/scratch"
      data: "/data"

Exclude Patterns

defaults:
  zip:
    exclude:
      - ".git/"
      - "node_modules/"
      - "__pycache__/"
      - ".venv/"
      - "*.tmp"
      - ".DS_Store"
      - "target/"

Examples

Homelab file transfers

Configure your homelab servers once, then quickly move files between machines.

# Copy ML model to GPU workstation
qc file model.pt andromeda:scratch

# Push code directory to build server
qc dir saturn:projects --extract

# Pull results back from server
qc pull andromeda:results

Development workflows

Push code to remote build servers, pull results back.

# Push current project to remote
qc dir build-server:workspace --extract

# Pull built artifacts
qc pull build-server:dist

# Preview before sending
qc --dry-run dir production:deploy

Backup automation

Script regular dumps to shared storage with automatic organization.

# Dump current directory to NAS
qc dump

# Dump specific file to shared storage
qc dump important-data.db

# Dump to custom share
qc dump ./project --to backup-nas:archives

Multi-machine projects

Work on projects across different machines with quick directory sync.

# Send local changes to remote workstation
qc dir workstation:current-project --extract

# Pull updates from remote
qc pull workstation:current-project

# Check what would be sent
qc --dry-run dir workstation:current-project

Global Options

Option Description
-c, --config <path> Custom config file
-v, --verbose Show ssh/scp commands
--json Output in JSON format
--dry-run Preview operations without executing
